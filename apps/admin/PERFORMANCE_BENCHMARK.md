# ⚡ 성능 벤치마크 가이드

## 📊 업로드 속도 비교

### 준비물
- 100MB 영상 파일
- 스톱워치 또는 타이머

---

## 테스트 1: 기존 시스템 (TUS OFF)

### Step 1: Feature Flag 비활성화

`.env.local` 파일 수정:
```bash
NEXT_PUBLIC_USE_TUS_UPLOAD=false
NEXT_PUBLIC_USE_DIRECT_FILE_UPLOAD=false
```

### Step 2: 서버 재시작
```bash
# Ctrl + C
npm run dev
```

### Step 3: 업로드 테스트
1. 브라우저 새로고침
2. 100MB 파일 선택
3. 스톱워치 시작
4. 업로드 완료 시 정지
5. **시간 기록**: ___초

---

## 테스트 2: 개선된 시스템 (TUS ON)

### Step 1: Feature Flag 활성화

`.env.local` 파일 수정:
```bash
NEXT_PUBLIC_USE_TUS_UPLOAD=true
NEXT_PUBLIC_USE_DIRECT_FILE_UPLOAD=true
```

### Step 2: 서버 재시작
```bash
npm run dev
```

### Step 3: 업로드 테스트
1. 브라우저 새로고침
2. **같은 100MB 파일** 선택
3. 스톱워치 시작
4. 업로드 완료 시 정지
5. **시간 기록**: ___초

---

## 📈 결과 비교

| 방식 | 시간 (초) | 개선율 |
|------|----------|--------|
| 기존 (Direct Upload) | ___ | - |
| 개선 (TUS Protocol) | ___ | ___% |

**계산식:**
```
개선율 = ((기존 - 개선) / 기존) × 100
```

**예상 결과:**
- 기존: 180초
- 개선: 60초
- 개선율: **67%** ⬇️

## 🔍 추가 측정 항목

### 메모리 사용량

**측정 방법:**
1. Chrome Task Manager (Shift + Esc)
2. 업로드 중 메모리 확인

| 방식 | 메모리 (MB) | 개선율 |
|------|-----------|--------|
| 기존 | ___ | - |
| 개선 | ___ | ___% |

**예상:**
- 기존: 500MB
- 개선: 200MB
- 개선율: **60%** ⬇️

---

### 네트워크 데이터 전송량

**측정 방법:**
1. Chrome DevTools (F12) > Network 탭
2. Clear 버튼 클릭
3. 업로드 시작
4. 하단 "transferred" 값 확인

| 방식 | 전송량 (MB) | 개선율 |
|------|-----------|--------|
| 기존 (Base64) | ___ | - |
| 개선 (Direct) | ___ | ___% |

**예상 (100MB 파일):**
- 기존: 133MB (Base64 인코딩)
- 개선: 100MB
- 개선율: **25%** ⬇️
